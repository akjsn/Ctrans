Checking Assignments

Spring 2019: Question 27 (use C language to make this program)

A teaching assistant places assignments that need to be checked by putting them on a stack. This means that the assignment placed at the bottom of the stack will be the last assignment to be checked. If a student comes to the teaching assistant to hand in that assignment (note that the submission is already considered late at this point), that assignment will be placed on the top of the stack, therefore, the assignment which was just handed in by the student will be the next assignment to be checked.

Of course, a day in the life of a teaching assistant is not dedicated completely to checking assignments -- a teaching assistant still has other things to do, such as attending classes, working on projects, and what not. 助教辛苦了！ Therefore, the teaching assistant may not be able to finish correcting assignments in one day. At the end of the day, when the teaching assistant signs out, find out whose assignments have not been checked.

Use the following code to define the structure of an assignment.

typedef struct Homework{
    char name[10]; //student name
    int isDelay; //if submitted late then True, else False
    struct Homework *next;
}HW;

Input instructions:

  1. First, input a series of names, one at a time. Names entered first represent assignments that are submitted earlier, and are placed at the bottom of the stack.
  2. Input -1 to end the abovementioned input.
  3. Next, there are three functionss:
    • Input 1, followed by the student name if there is a late submission. This will add the assignment to the top of the queue. Remember to mark this assignment as a late submission.
    • Input 2 to correct the assignment of one student (i.e. remove one assignment from the top of the stack).
    • Input 3 to sign out.

Output instructions:

  1. For Function 2, output the name of the student whose assignment was corrected, and also output whether the assignment was submitted late or not.
  2. For Function 3, output all the student names whose assignments have not been corrected, and also output whether the assignment was submitted late or not.
  3. If, when 2 or 3 is input, there are no assignments in the stack, output "Empty".
