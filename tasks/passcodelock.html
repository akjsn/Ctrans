Passcode Lock

Fall 2018: Question 23 (use Python to write this program)
Fall 2017: Question 13 (use C language to write this program)

Nowadays, we frequently use a passcode lock to keep our valuables safe, however, if we forget the password for that, then it would be catastrophic!

If this happens on a traditional passcode lock (with 4 digits), we will have to try at most 10,000 (104) times before the lock is opened, so we will have to design a system to find out the amount of correct numbers in the password we entered. Using this method, unlocking can become faster.

Design a program to determine how many correct numbers are needed before the lock is unlocked.

Input instructions:

  1. This program allows users to input multiple times.
  2. First, input 4 numbers, each from 0 to 9 (separated with a space); this number represents the correct password.
  3. For every input afterward, input 4 numbers per line (also from 0 to 9, separated with a space); which represents the number to be tested. One line represents a password to be tested.
  4. An input of -1 terminates the program.

Output instructions:

  1. Output has to be in this format: pAqB, from which:
    • p represents the correct numbers which are in the correct positions
    • q represents the correct numbers which are NOT in the correct positions
  2. Best result rule: If there are repeated numbers, then the repeated numbers can only be counted once.
    For example: If the correct password is 5543, and the user guesses 5255, do not use the first “5” in the guess for comparison against the second number in the correct password; therefore, using the best result rule, the output after comparison has to be 1A1B, from which the first “5” is correct, therefore the result is counted as 1A, however, only one of the “5” (either the third, or the fourth) matches the second “5” in the correct password, therefore the result in this case is counted as 1B. If, in this case, the user guesses 5267, the first “5” in the guess cannot be compared against the second “5” in the correct password, therefore the output is 1A0B.
