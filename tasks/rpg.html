An RPG Game

Spring 2018: Question 31 (use C language to write this program)

Johann wants to make an RPG game. The RPG game consists of a fighter, weapons and monsters in a 100 x 100 battlefield.

  • A fighter has the following stats: HP, attack power, weapons, and position.
    The fighter has to attack monsters. The fighter can change weapons during the game.
  • A weapon has the following stats: name and attack range.
  • A monster has the following stats: ID, HP, attack power, and position.

Before the game can start, the player must build the game setting. After the game has started, the player CANNOT add any more fighters, monsters, or weapons.

On a fighter’s turn, a fighter can do any of the following actions:

  • attack (a)
  • move (m)
  • equip (equip)
A fighter can also look at the character data (d) without losing his turn.

Once it is the monsters’ turn, if a fighter is within the attack range of a monster, the monster will attack.

Game rules:

  1. Damage inflicted to the opponent is as follows:
    Attack Power Damage
    0-10 5
    11-50 10
    51 and above 20
  2. A fighter gains 5 strength for every monster killed.
  3. At the start of the game, the fighter starts with the fist as his weapon, with an attack range of 1. A monster’s attack range is 2.
  4. The fighter, on his turn, can choose to attack, move, or change weapon. The fighter can only move a maximum of 5 spaces in one axis (left-right, up-down), and there must not be any obstacle on the destination space. Once the fighter’s turn ends, the monsters will attack.
  5. Monsters can attack a fighter if the fighter is within the monster’s attack range; but monsters cannot move.
  6. There can only be ONE fighter; but there can be MANY monsters.
  7. The size of the battlefield is 100 x 100.
  8. When building the game setting, commands used are as follows:
    • is: create fighter
    • im: create monster
    • iw: create weapon
    • start: start game
  9. During the game, commands used are as follows:
    • a [monster ID]: attack monster [monster ID]
    • m [position]: move to [position]
    • equip [weapon name]: equips weapon [weapon name]
    • d: shows the fighter’s stats, then (in order of ID) the monsters’ stats.
  10. If there is no fighter or monster, then the game ends. Output the winner and the stats (redo the function of d in rule 9)
  11. Output “Error!” for any invalid input. If the input is invalid during a turn, the turn will not end until a valid input is given.
  12. A fighter or monster dies when the HP falls to 0 or lower.
  13. At the start of the game, ID, HP, attack power, and attack range are all above 0.

Input instruction: Input format is as follows:

  1. When creating game setting:
    • create fighter (is): is [HP] [attack power] [position-row] [position-column]
    • create weapon (iw): iw [weapon name] [attack power]
    • create monster (im): im [ID] [attack power] [position-row] [position-column]
  2. In-game:
    • attack (a): a [monster ID]
    • move (m): m [position-row] [position-column]
    • equip: equip [weapon name]
    • view character data (d): d
