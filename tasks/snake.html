Snake Game

Spring 2018: Question 37 (use C language to write this program)

Using a linked list, simulate the gameplay of the snake game we all have ever played in older mobile phones.

Game rules:

  1. At the start of the game, the snake has an initial length of 5.
  2. The game field is quadrilateral, and has a size of 10 x 10, and each space in the game field is marked, from (0, 0) to (9, 9), as follows:
  3. At the start of the game, the snake’s head is at (4, 0), and the snake’s tail is at (0, 0).
  4. The system must decide if the move input is legal; and the game can continue if the input is legal. Illegal moves are defined in Point 6 of the game rules.
  5. The snake’s body will grow by 1 if it eats a food item.
  6. If any of the following occur, then the move becomes illegal (and there will be no position changes):
    • snake goes out of bounds (x < 0 or y < 0 or x > 9 or y > 9)
    • snake touches any part of its own body
      For example: When the snake’s tail is at (7, 2) and the snake’s head is at (7, 1), the snake wants to move down to (7, 2) – which is not possible.

Input instruction: For each line of input, input the coordinates (x, y) for the location of the food item, and the movements made by the snake (one character represents the snake moving one space in the direction given – u (up), d (down), l (left), r (right)), all separated with a space.

Output instructions:

  1. If the snake fails to reach the food item, output “Fail”.
  2. If the snake reaches the food item, output “Success”.
  3. If any of the situations described in point 6 of the game rules (illegal moves) occur, output “Game Over” and end the program.

Sample Input Expected Output
7 0 rr
7 0 r
4 2 ddlll
6 1 urr
7 4 rdd 		Fail
Success
Success
Success
Game Over
